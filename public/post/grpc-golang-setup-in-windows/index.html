  1. 下载protoc，打开链接 下载后将对应的文件解压到gopath的bin目录。
  2. 下载protoc的golang插件。下载地址 链接 下载后放在protoc的同级目录（需要改扩展名为exe）

syntax = "proto3";
option go_package = ".;hello";
package main;

 message String {
     string value = 1;
}

注意:

参考官网的步骤，精简下安装流程

  1. 启用GOMOUDLE 运行 set GO111MODULE=on (临时)或者设置环境变量

  2. 安装生成相关工具

    go插件

go get -u google.golang.org/protobuf/cmd/protoc-gen-go 
go get -u google.golang.org/grpc/cmd/protoc-gen-go-grpc

protobuf编译器 可以去github release页面下载，我这里直接给出地址，请按系统架构选择，下载解压到GOPATH目录即可

https://github.com/protocolbuffers/protobuf/releases/download/v3.14.0/protoc-3.14.0-win32.zip
https://github.com/protocolbuffers/protobuf/releases/download/v3.14.0/protoc-3.14.0-win64.zip

vsCode可以安装插件 vscode-proto3 后，在 settings.json 中添加下面的内容，将可以实现保存proto文件后自动生成对应的go文件：

"protoc": {

        "path": "${env.GOPATH}/bin/protoc",
        "compile_on_save": true,
        "options": [
            "--proto_path=.",
            "--proto_path=${workspaceRoot}/proto",
            "--experimental_allow_proto3_optional",
            // "--csharp_out=${workspaceRoot}/proto",
            "--go_out=${workspaceRoot}/proto",
            "--go_opt=paths=source_relative",
            "--go-grpc_out=require_unimplemented_servers=false:${workspaceRoot}/proto",
            "--go-grpc_opt=paths=source_relative"
        ]
    },

注：#

处理官方库不能访问的问题

go mod edit -replace=google.golang.org/grpc=github.com/grpc/grpc-go@latest
go mod tidy
go mod vendor
go build -mod=vendor

grpc生态插件 github

go install \
    github.com/grpc-ecosystem/grpc-gateway/v2/protoc-gen-grpc-gateway \
    github.com/grpc-ecosystem/grpc-gateway/v2/protoc-gen-openapiv2 \
    google.golang.org/protobuf/cmd/protoc-gen-go \
    google.golang.org/grpc/cmd/protoc-gen-go-grpc

常见问题#

  • protoc-gen-go failed :: The import path must contain at least one forward slash (’/’) character."

     修改proto定义中的 `option go_package="pb";` 为 `option go_package = "./;pb";`
